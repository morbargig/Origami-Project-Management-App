# Delivery Flow

## Operating Model

Origami Smart PM supports this operating sequence:

1. Define context.
2. Set the goal.
3. Break work into Project, Epic, and Task.
4. Execute through the task and epic boards.
5. Validate through Self QA and formal epic QA.
6. Deliver with management visibility on progress, dates, and budget.

## Accountability Model

- `Project Manager` owns management visibility.
- `Delivery Manager` runs day-to-day delivery.
- `Owner` drives epic-level delivery.
- `Assignee` owns actual task completion.
- `QA Manager` and `QA Team` manage formal routed QA.
