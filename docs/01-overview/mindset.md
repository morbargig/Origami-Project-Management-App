# Mindset

## Core Delivery Sequence

Context -> Goal -> Plan -> Execution -> QA -> Delivery

## Product Structure

Project -> Epic -> Task

## Management Principles

Every major part of the system maps directly to a project-management need:
- `Business Context` explains why the initiative exists
- `Project Goal` defines what success looks like
- dates and Gantt views support planning
- delivery and QA assignments support accountability
- budget fields support cost control
- QA flow supports quality control
- dashboards support governance and visibility

## Separation of Concerns

- `PRD.md` defines why the system exists and what it contains
- `BUILD_GUIDE.md` defines how to configure it in Origami
- `docs/04-views/` describes user-facing pages
- `docs/05-operations/` describes how teams should run the model

## Naming Consistency

- `System Managers`
- `Project Managers`
- `Delivery Manager`
- `Delivery Team`
- `QA Manager`
- `QA Team`
- `Owner`
- `QA Owner`
- `Assignee`
- `QA Assignee`
- `Self QA`
- `Ready for QA`
