# Decision Log

## Purpose

This log records key documentation and governance decisions for `JiraGami`.

- rationale: the application is already in production and the goal is documentation maturity, not product redesign

- rationale: this keeps the project identity clear while still describing `Origami` as the deployment platform

### Decision 03

- decision: keep the existing numbered `docs/` structure and extend it with a governance layer
- rationale: the current structure is already understandable and only needs stronger project-management coverage

- rationale: they already capture the product and implementation truth and should not be replaced by summary documents
