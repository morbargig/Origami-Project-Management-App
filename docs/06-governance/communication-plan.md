# Communication Plan

## Purpose

This communication plan defines how `JiraGami` information should be shared across management, delivery, implementation, QA, and stakeholder audiences.

## Communication Principles

- communicate from canonical sources whenever product truth is involved
- keep management summaries short, decision-oriented, and traceable to source documents
- avoid copying product truth into multiple places unnecessarily
- use dashboard, workflow, budget, and governance documents as reporting references

## Audience And Cadence

### Project Manager and Delivery Manager

- cadence: weekly or as needed
- content: roadmap, RAID review, delivery progress, blockers, next actions

### Product and implementation leads

- cadence: weekly
- content: PRD alignment, configuration impact, workflow implications, documentation changes

### Quality Assurance (QA) leads

- cadence: weekly or per release milestone
- content: QA flow, open validation items, routing issues, readiness status

### Stakeholders and clients

- cadence: milestone-based or monthly
- content: high-level status, scope position, roadmap progress, key risks, decisions
