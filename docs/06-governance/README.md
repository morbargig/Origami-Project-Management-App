# Governance

This folder contains the project management control layer for `JiraGami`, the documentation framework for `Origami Smart PM`.

## Purpose

Use this section to manage the production project professionally without changing the underlying product truth.

These documents define:
- why the project exists from a management perspective
- what is in scope and out of scope
- what risks and dependencies require attention
- who is responsible for delivery and decision-making
- how stakeholders should be informed
- how management success should be measured

## Files

- [project-charter.md](project-charter.md) - executive project framing and objectives
- [scope-baseline.md](scope-baseline.md) - in-scope, out-of-scope, assumptions, and constraints
- [roadmap.md](roadmap.md) - management roadmap and delivery themes
- [raid-log.md](raid-log.md) - risks, assumptions, issues, and dependencies
- [raci.md](raci.md) - responsibility and accountability matrix
- [communication-plan.md](communication-plan.md) - stakeholder communication model
- [decision-log.md](decision-log.md) - documented management and documentation decisions
- [success-metrics.md](success-metrics.md) - management success measures and reporting logic

## Governance Rule

These files support planning, governance, and communication.

They do not change:
- the Product Requirements Document (PRD)
- the implementation model
- production field names
- production workflows
- documented page logic
