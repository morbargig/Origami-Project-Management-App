# Project Management

This folder contains the professional project management document set for `JiraGami`.

`JiraGami` is the project name.

The product is a Smart Project Management system deployed on the `Origami` platform and influenced by Jira-style working practices.

## Purpose

Use this section to manage the project professionally without changing the underlying product truth.

These documents define:
- the project identity
- the project management framing
- scope and control boundaries
- reporting and communication standards
- risks, assumptions, issues, and dependencies
- management accountability

## Files

- [PDR.md](PDR.md) - Project Definition Report (PDR)
- [project-charter.md](project-charter.md) - executive project framing and objectives
- [scope-baseline.md](scope-baseline.md) - in-scope, out-of-scope, assumptions, and constraints
- [roadmap.md](roadmap.md) - management roadmap and delivery themes
- [raid-log.md](raid-log.md) - risks, assumptions, issues, and dependencies
- [raci.md](raci.md) - responsibility and accountability matrix
- [communication-plan.md](communication-plan.md) - stakeholder communication model
- [project-status-report.md](project-status-report.md) - professional status reporting format
- [decision-log.md](decision-log.md) - documented management and documentation decisions
- [success-metrics.md](success-metrics.md) - management success measures and reporting logic

## Governance Rule

These files support planning, governance, reporting, and communication.

They do not change:
- the Product Requirements Document (PRD)
- the implementation model
- production field names
- production workflows
- documented page logic
