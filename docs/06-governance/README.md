# Project Management

This folder contains the governance and reporting layer for `JiraGami`.

## Purpose

Use this section to manage the project professionally while keeping the live product model unchanged.

These documents define:
- project framing
- scope and control boundaries
- reporting and communication standards
- risks, assumptions, issues, and dependencies
- accountability and management decisions

## Files

- [PDR.md](PDR.md) - executive project definition and governance framing
- [project-charter.md](project-charter.md) - objectives, scope, and stakeholder framing
- [scope-baseline.md](scope-baseline.md) - in-scope, out-of-scope, assumptions, and constraints
- [roadmap.md](roadmap.md) - management roadmap and documentation maturity direction
- [raid-log.md](raid-log.md) - risks, assumptions, issues, and dependencies
- [raci.md](raci.md) - responsibility and accountability matrix
- [communication-plan.md](communication-plan.md) - stakeholder communication model
- [project-status-report.md](project-status-report.md) - status reporting template
- [decision-log.md](decision-log.md) - key project and documentation decisions
- [success-metrics.md](success-metrics.md) - governance and visibility success measures

## Governance Rule

These files support planning, governance, reporting, and communication.

They do not change:
- the Product Requirements Document (PRD)
- the Build Guide
- the page and view baseline
- field names
- workflow names
- documented page logic
